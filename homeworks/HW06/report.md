## 1. Dataset

- Какой датасет выбран: `S06-hw-dataset-01.csv`
- Размер: (12000, 30)
- Целевая переменная: `target` (0: 67.66%, 1: 32.34%)
- Признаки: 24 числовых (float64), 5 целочисленных (int64), которые могут интерпретироваться как категориальные (cat_contract, cat_region, cat_payment, tenure_months). Столбец `id` исключен из признаков.

## 2. Protocol

- Разбиение: train/test (75/25, `random_state=42`, стратификация по `target`)
- Подбор: GridSearchCV на train с использованием 2-фолдовой кросс-валидации (StratifiedKFold). Оптимизировалась метрика `roc_auc`.
- Метрики: accuracy, F1, ROC-AUC. ROC-AUC является основной метрикой, так как она устойчива к дисбалансу классов и оценивает способность модели разделять классы по вероятностям.

## 3. Models

Описаны следующие модели:

- **DummyClassifier**: baseline, стратегия `most_frequent`.
- **LogisticRegression**: baseline, использовался Pipeline с `StandardScaler`.
- **DecisionTreeClassifier**: базовая модель, подбирались `max_depth` (10, 15) и `min_samples_leaf` (15, 20).
- **RandomForestClassifier**: ансамбль деревьев, подбирались `max_features` (3, 5, 10, 15, 20) и `min_samples_leaf` (1, 2, 3, 5, 10).
- **AdaBoostClassifier**: бустинг над деревьями решений.
- **StackingClassifier**: композиция DT, RF и AdaBoost с логистической регрессией в качестве мета-модели.

## 4. Results

### Финальные метрики на test:

| Модель | Accuracy | F1-score | ROC-AUC |
| :--- | :---: | :---: | :---: |
| Dummy | 0.677 | - | - |
| Logistic Regression | 0.829 | 0.713 | 0.784 |
| Decision Tree (tuned) | 0.779 | 0.659 | 0.748 |
| Random Forest (base) | 0.918 | 0.863 | 0.887 |
| AdaBoost | 0.861 | 0.770 | 0.824 |
| **Random Forest (tuned)** | **0.970** | **0.890** | **0.911** |
| Stacking | 0.921 | 0.876 | 0.906 |

Победитель: **RandomForestClassifier (tuned)** с ROC-AUC = 0.911. Модель показала наилучшую обобщающую способность после подбора гиперпараметров.

## 5. Analysis

- **Устойчивость**: Использование фиксированного `random_state` обеспечивает воспроизводимость. При небольших изменениях разбиения метрики ансамблей (RF, Stacking) остаются более стабильными по сравнению с одиночным деревом.
- **Ошибки**: Судя по confusion matrix, модель хорошо справляется с обоими классами, однако есть небольшое количество ложноположительных и ложноотрицательных срабатываний, что ожидаемо при ROC-AUC ~0.91.
- **Интерпретация**: Permutation importance показал, что наиболее важными признаками являются `num19`, `num18` и `num07`. Эти признаки вносят наибольший вклад в предсказательную способность модели.

## 6. Conclusion

1. Ансамбли (Random Forest, Stacking) значительно превосходят одиночные деревья и линейные модели на данном датасете.
2. Подбор гиперпараметров (GridSearchCV) критически важен для достижения высоких метрик, особенно для Random Forest.
3. Честный ML-протокол (отложенная тестовая выборка, CV на train) позволяет получить адекватную оценку качества модели и избежать переобучения.
4. Деревья решений чувствительны к параметрам сложности (глубина, количество объектов в листе), и без их контроля быстро переобучаются.
